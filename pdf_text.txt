What Is DispatchDZ?
The Problem We Solve
Right now in Algeria, if a restaurant owner needs tomatoes, he calls 7 suppliers one by one every morning.
Half don't answer. Half say ma3anash (we don't have it). He wastes 2 hours — every single day.
This is not just restaurants. A contractor needs cement — calls 4 depots. A mechanic needs a spare part —
calls 5 wholesalers. A grocery owner needs to restock — calls 3 distributors. Every B2B purchase in Algeria
starts with a chain of phone calls.
 ■
The Core Idea
Instead of buyers chasing suppliers one by one, suppliers broadcast what they have — and buyers get
instant ranked answers. Like a live feed for B2B stock. Supplier updates in 30 seconds. Buyer finds a
match in 10 seconds.
The 4 Sectors at Launch
Sector
Who Uses It
■ Food & Produce
Restaurants, cantines, hotels, fast food chains — daily stock needs, highest urgency
■ Construction
Contractors and builders needing cement, iron, tiles, cables
■ Auto Parts
Garages, mechanics, fleet managers — spare parts by brand and model
■ General Wholesale
Épiceries, superettes restocking from distributors
The Two Products — Clear Separation
 ■
The Website = Trust & Acquisition ONLY
A marketing presentation. Explains what DispatchDZ is, who it is for, and how it works. Its ONLY goal is to
make a visitor download the app. No product functionality here. Think of it as the public face — like
Airbnb.com.
 ■
The Mobile App = Where EVERYTHING Happens
The actual product. Suppliers broadcast stock. Buyers search and find matches. Alerts fire. Price
comparisons appear. Analytics live here. Both supplier and buyer roles are inside the SAME app — the
interface adapts based on the user's chosen role.

System Architecture
Read this section before writing a single line of code. It explains how every part of DispatchDZ fits together.
 ■
■
Why Supabase Is the Cornerstone
Supabase replaces: a Node.js API server, a PostgreSQL server, a WebSocket server, a file storage
server, and an auth service — all in one platform. 5 separate things your team does NOT need to build
from scratch.
The 4 Core Data Flows
Flow 1 — Supplier Broadcasts Stock
   Supplier fills the form in the app and taps 'Broadcast'
   App inserts a new row into the stock_listings table in Supabase
   Supabase Realtime fires immediately — all connected buyers see the new card appear
   Buyers with saved search alerts receive a push notification on their phone
Flow 2 — Buyer Searches
   Buyer types a query (or speaks using voice search)
   App calls the search_listings custom SQL function in Supabase
   Function ranks results by: text match quality + proximity (same wilaya) + quantity
   Results include market average price and cheapest option highlighted
Flow 3 — Shortage Alert (Automatic, every 30 min)
   A Supabase Edge Function runs automatically every 30 minutes
   It counts active listings per product category per wilaya
   If any combination drops below 3 suppliers — a shortage alert is created
   All buyers in that wilaya and sector receive a push notification immediately
Flow 4 — WhatsApp Bot
   User sends a WhatsApp message in French, Arabic, or Darija
   Meta sends it to the webhook at /api/whatsapp on Vercel
   GPT-4o-mini parses intent and extracts: product, quantity, wilaya
   search_listings runs and top 3 results are sent back as a WhatsApp reply

This SQL function is the heart of the product. It runs every time a buyer searches. Copy it exactly into a new
Supabase SQL function.
CREATE OR REPLACE FUNCTION search_listings(
  query      text,
  p_sector   int,
  p_wilaya   int,
  p_qty      numeric DEFAULT 0
)
RETURNS TABLE (id uuid, product_name text, price_per_unit numeric,
  quantity numeric, unit text, supplier_name text,
  supplier_phone text, wilaya_name text,
  delivery_available boolean, avg_market_price numeric,
  lowest_price numeric, match_score real)
AS $$
  WITH market_stats AS (
    SELECT AVG(price_per_unit) AS avg_price,
           MIN(price_per_unit) AS min_price
    FROM stock_listings
    WHERE status = 'active' AND sector_id = p_sector
      AND product_name ILIKE '%' || query || '%'
  )
  SELECT sl.id, sl.product_name, sl.price_per_unit,
    sl.quantity, sl.unit, u.business_name AS supplier_name,
    u.phone AS supplier_phone, w.name_fr AS wilaya_name,
    sl.delivery_available, ms.avg_price, ms.min_price,
    -- Ranking: text match x3 + same wilaya +2 + qty match +1
    (similarity(sl.product_name, query) * 3
     + CASE WHEN sl.wilaya_id = p_wilaya THEN 2 ELSE 0 END
     + CASE WHEN sl.quantity >= p_qty THEN 1 ELSE 0 END
    ) AS match_score
  FROM stock_listings sl
  JOIN users u ON sl.supplier_id = u.id
  JOIN wilayas w ON sl.wilaya_id = w.id
  CROSS JOIN market_stats ms
  WHERE sl.status = 'active' AND sl.sector_id = p_sector
    AND sl.product_name ILIKE '%' || query || '%'
    AND sl.expires_at > now()
  ORDER BY match_score DESC LIMIT 5;
$$ LANGUAGE sql;

WhatsApp Bot
A lightweight access layer for buyers who want quick answers without opening the app. Lives as a single API
route inside the Next.js project.
How It Works — Step by Step
1.   User sends a WhatsApp message in French, Arabic, or Darija
2.   Meta sends it to the webhook at /api/whatsapp on Vercel
3.   GPT-4o-mini parses the message and extracts: intent (search/help/register), product name, quantity,
wilaya
4.   Supabase search_listings runs with those parameters and returns the top 3 results
5.   Results formatted into a clean WhatsApp reply: supplier name, price, quantity, phone number
6.   Reply sent back via Meta Cloud API

Intelligence Layer
Shortage Detection — Runs Every 30 Minutes Automatically
1.   Counts active listings per product category per wilaya across all 58 wilayas
2.   If any combination has fewer than 3 active suppliers — a shortage alert is created
3.   All buyers in that wilaya and sector receive a push notification on their phone
4.   When stock recovers above 3 suppliers — the alert is automatically resolved

import { createClient } from '@supabase/supabase-js'
const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
)
async function detectShortages() {
  const { data } = await supabase.rpc('count_active_by_category_wilaya')
  for (const row of data) {
    if (row.active_count < 3) {
      await supabase.from('shortage_alerts').upsert({
        wilaya_id: row.wilaya_id,
        sector_id: row.sector_id,
        category_id: row.category_id,
        active_supplier_count: row.active_count,
        status: 'active',
        triggered_at: new Date().toISOString()
      }, { onConflict: 'wilaya_id,category_id' })
      await notifyBuyers(row.wilaya_id, row.sector_id, row.category_name)
    }
  }
}
async function notifyBuyers(wilayadId, sectorId, categoryName) {
  const { data: buyers } = await supabase
    .from('users').select('expo_push_token')
    .eq('wilaya_id', wilayadId)
    .contains('sector_ids', [sectorId])
    .not('expo_push_token', 'is', null)
  const messages = buyers.map(b => ({
    to: b.expo_push_token,
    title: `Penuries: ${categoryName}`,
    body: 'Stock critique dans votre wilaya. Commandez maintenant.',
    data: { screen: 'alerts' }
  }))
  await fetch('https://exp.host/--/api/v2/push/send', {
    method: 'POST', body: JSON.stringify(messages)
  })
}

Day One Checklist — Start Here
When your team meets for the first time, do these 4 tasks in parallel. By end of day one, all three codebases
exist and all talk to the same backend.
Backend Person
1.   Go to supabase.com — create a new project — save the URL and API keys in a shared .env file
2.   Create all tables using the schema from Section 03 of this document
3.   Enable Row Level Security on all tables immediately
4.   Seed the wilayas table (58 rows), sectors (4 rows), and product_categories
5.   Share the .env file securely with all team members
Mobile Person
1.   Run: npx create-expo-app dispatchdz-mobile --template blank-typescript
2.   Install: expo-router, nativewind, @supabase/supabase-js, i18next, react-i18next
3.   Connect the Supabase client using the shared .env credentials
4.   Build Screen 1 of onboarding (language picker) to confirm the setup works end to end
Web Person
1.   Run: npx create-next-app dispatchdz-web --typescript --tailwind --app
2.   Install: @supabase/supabase-js, shadcn/ui, next-intl
3.   Connect the Supabase client using the shared .env credentials
4.   Deploy the empty project to Vercel to confirm CI/CD from GitHub works
Product / QA Person
1.   Open Figma — create the shared design file for the team
2.   Define the color palette: Primary Green #1A5C38, Dark #1C1A14, and sector accent colors
